# elle-plugin

Stable ABI crate for Elle plugins. Zero dependencies.

## Purpose

Defines the types and macros needed to write Elle plugins that are
ABI-stable. Plugins depend on this crate (not on `elle`), so they
can be compiled independently and loaded at runtime.

## ABI Design

Single resolve function — same pattern as `vkGetInstanceProcAddr`.
Plugins look up API functions by name at init time. Adding functions
to elle never breaks existing plugins.

## Key Types

| Type | Purpose |
|------|---------|
| `ElleValue` | Opaque 16-byte value (same layout as elle's `Value`) |
| `ElleResult` | Primitive return: `{ signal: u32, value: ElleValue }` |
| `ElleApiLoader` | Resolve function + version — the entire ABI contract |
| `Api` | Resolved function pointer cache (generated by `elle_api!`) |
| `EllePrimDef` | Primitive metadata (C-compatible) |
| `EllePrimFn` | `extern "C" fn(args, nargs) -> ElleResult` |
| `EllePluginCtx` | Registration context for `elle_plugin_init` |
| `ElleKV` | Key-value pair for struct construction |

## Macros

| Macro | Purpose |
|-------|---------|
| `elle_api!` | Generates `Api` struct and `Api::load()` from function declarations |
| `define_plugin!` | Generates `elle_plugin_init` entry point + `api()` accessor |

## Signal Constants

`SIG_OK`, `SIG_ERROR`, `SIG_YIELD`, `SIG_IO`, `SIG_FFI`, etc.

## Invariants

1. This crate has zero dependencies — types only.
2. All types are `#[repr(C)]` for ABI stability.
3. `ElleValue` is never inspected — always accessed through `Api`.
4. Plugin .so is never unloaded — string pointers from rodata are static.
